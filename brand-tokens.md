# brand-tokens.md — 品牌 Token 映射表 v3.0

> **用途：** 定义品牌色对 shadcn CSS 变量的覆盖。
> **原则：** 只覆盖有品牌化需求的 token，其余全部沿用 shadcn 默认值。
> **唯一配色体系：** shadcn HSL CSS Variables。不再使用 `--fx-*` 前缀。

---

## 品牌色覆盖（3 个变量）

| 用途 | shadcn CSS 变量 | HSL 值 | Hex 等价 |
|------|----------------|--------|---------|
| 主色（主按钮/主操作/激活态） | `--primary` | `30 100% 50%` | `#FF8000` |
| 主色前景（主按钮文字） | `--primary-foreground` | `0 0% 100%` | `#FFFFFF` |
| 焦点环 | `--ring` | `30 100% 50%` | `#FF8000` |

**品牌浅色（通过 accent 映射）：**
- `--accent: 37 100% 94%` (#FFF7E6)
- `--accent-foreground: 30 100% 50%` (#FF8000)

**其余 token 不覆盖，沿用 shadcn 默认值。**

## ⚠️ 无障碍合规注记（2026-07-22 实测；默认值未改，覆盖责任在项目侧）

本表默认的 `*-foreground: #FFFFFF`（白字）与上述亮底色搭配，**实测全部不满足 WCAG 2.x AA**
（正文 4.5:1 / 非文本 3:1）。按 WCAG 相对亮度公式实算，可复现：

| 组合 | 白字对比度 | AA 正文 | 改用近黑 `#181C25` |
|---|---|---|---|
| `--primary` `#FF8000` | **2.52** | ✗ | **6.77** ✓ |
| `--info` `#189dff` | 2.87 | ✗ | 5.93 ✓ |
| `--success` `#87cc3b` | **1.96** | ✗ | 8.71 ✓ |
| `--warning` `#ff7c19` | 2.58 | ✗ | 6.62 ✓ |
| `--destructive` `#ff4a66` | 3.27 | ✗（过非文本 3:1） | 5.22 ✓ |
| `--accent-foreground` `#FF8000` on `--accent` `#FFF7E6` | 2.36 | ✗ | — |
| **次要文字 `#91959E`（`--fx-n11`/`.text-n11`）on `#FFFFFF`** | **3.00** | ✗ | **`#6E727B` = 4.82** ✓ |

> **⚠️ 次要文字色是同源问题的第二面（2026-07-22 追加，此前遗漏）：** 上表最后一行不是"白字配亮底"，
> 而是**深色文字配白底**——但同样不合规。`#91959E` 被 `office/SKILL.md` 的四级字号表指定为
> **L3 辅助标签/字段名** 与 **L4 弱信息/时间戳**，那是**真实文字**、适用正文 4.5:1，而它只有 3.00:1。
> **处置：`#91959E` 降级为仅供非文字装饰**（分隔点 / 图标 / 滚动条，3:1 足够）；**凡承载文字的槽位改用
> 同色相压深的 `#6E727B`（4.82:1）**。此替换已由 Open Design 在 2026-07-22 的原型里自行做出并附回退注释，
> 可作实证参考。

**为什么默认值不改：** 这套色板是为国内市场语境建立的，白字亮底是国内 UI 通行做法；
改默认值会改变所有既有原型与母版产出的观感。**这不是当初做错，是新语境（国际化/出海）
让它成为承重项。**

**什么时候必须覆盖（项目侧责任）：**
- 目标用户在**欧美市场**，或产品受 EAA（EU 2019/882，2025-06-28 生效，覆盖移动应用）/
  ADA（以 WCAG 2.1 AA 为诉讼基准）影响；纯 B2B 可能法律豁免，**但大客户采购条款会把它变成商业硬要求**
- 客户合同或招标文件写明无障碍要求

**两条已验算的合规解（任选其一，不必自己再算）：**
1. **保留品牌色值，改前景为近黑**（推荐）：`--primary-foreground` 由 `#FFFFFF` 改为 `#181C25`
   → 6.77:1。品牌色零改动。这也是欧美设计系统处理亮暖色的通行做法（IBM Carbon `warning #F1C21B`、
   Shopify Polaris `caution #FFE600` 均为深色字承载）。
2. **压深底色，维持白字**：品牌橙压至 `#B85C00`（同色相 H30/S100，L 50%→36%）→ 白字 4.60:1。
   代价是品牌色值改变。

暗色模式下 `#FF8000` 在 `#161616` 上为 **7.19:1**，作文字与图标可直接使用，无需调整。

> 溯源：2026-07-22 mobile-list 项目实测，见语义候选 `SC-20260722-003`。
> 同一注记已同步至 `.claude/skills/office/references/design-system-contract.md`。

## HTML 原型使用方式

所有母版已统一使用 `framework/shared-head.html` 中的配置。品牌色通过 `:root` 变量自动生效。

**在组件里使用：**
- 主操作按钮：`class="btn-default"` 或 `bg-primary text-primary-foreground hover:bg-primary/90`
- 激活态文字：`text-primary`
- 品牌浅色背景：`bg-accent text-accent-foreground`

**全页品牌色限制：≤3 处可见使用。**

---

## Figma 保险层使用方式

在 Figma Variables 面板里：
1. `color/primary` → `#FF8000`
2. `color/primary-foreground` → `#FFFFFF`
3. `color/ring` → `#FF8000`
