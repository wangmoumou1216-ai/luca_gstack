## Phase 1：Nielsen 10条可用性启发式

评分 0-4（4=完全符合，0=完全违反），每项发现违反项创建 finding：

| 编号 | 原则 | 检查要点 |
|------|------|---------|
| H1 | 系统状态可见性 | 加载/操作结果反馈是否及时？ |
| H2 | 系统与真实世界匹配 | 语言、图标是否直观符合用户习惯？ |
| H3 | 用户控制与自由 | 能否撤销/取消？误操作有无出路？ |
| H4 | 一致性和标准 | 相同操作在不同页面表现是否一致？ |
| H5 | 错误预防 | 危险操作有无确认步骤？ |
| H6 | 识别而非回忆 | 选项是否直接可见？用户需要记忆什么？ |
| H7 | 使用灵活性与效率 | 高频用户有无快捷方式？ |
| H8 | 审美和极简设计 | 有无与核心任务无关的干扰信息？ |
| H9 | 帮助用户识别和恢复错误 | 错误信息是否明确指向问题？ |
| H10 | 帮助文档 | 有无必要的帮助信息？ |

## Phase 2：Norman 7条设计原则

| 编号 | 原则 | 检查要点 | 评估 |
|------|------|---------|------|
| N1 | 预设用途 | 元素视觉是否暗示正确操作方式？ | ✅/⚠️/❌ |
| N2 | 意符 | 可交互元素是否有可视交互信号？ | ✅/⚠️/❌ |
| N3 | 约束 | 是否防止错误操作（禁用/隐藏不可用选项）？ | ✅/⚠️/❌ |
| N4 | 映射 | 控件位置是否与影响对象逻辑对应？ | ✅/⚠️/❌ |
| N5 | 反馈 | 每次操作后是否有即时反馈？ | ✅/⚠️/❌ |
| N6 | 概念模型 | 系统运作是否符合用户心理预期？ | ✅/⚠️/❌ |
| N7 | 可发现性 | 功能是否容易被发现？ | ✅/⚠️/❌ |

---

## Phase 3：WCAG 2.1 AA 关键检查

| 检查项 | 标准 | 判断 |
|-------|------|------|
| 颜色对比度（正文）| ≥ 4.5:1 | ✅/❌ |
| 颜色对比度（大文字）| ≥ 3:1 | ✅/❌ |
| 不只靠颜色传递信息 | 有其他区分方式 | ✅/❌ |
| 键盘可访问 | 所有功能键盘可完成 | ✅/❌/无法从截图判断 |
| 焦点可见性 | 键盘焦点有明显视觉标记 | ✅/❌/无法从截图判断 |
| 图片替代文本 | 功能性图片有 alt 文本 | ✅/❌/无法从截图判断 |
| 表单标签 | 每个输入都有关联标签 | ✅/❌ |
