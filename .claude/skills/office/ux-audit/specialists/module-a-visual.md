## 发现格式（每条必须包含所有字段）

```
### [UX-A-P{0/1/2}-{三位编号}] {标题}
- **位置：** {标准位置名}
- **发现：** {具体描述，指向具体元素}
- **违反规范：** {design-system-contract §X / 具体规则}
- **严重程度：** P0 / P1 / P2
- **优化方向：** {1句话，描述期望结果，不设计方案}
- **升级标注：** {是 — 违反 §X / 否}
```

**严重程度定义：**
- P0：严重阻碍用户完成核心 CRM 任务，必须修复
- P1：明显影响效率或可用性，应该修复
- P2：体验优化项，有余力时修复

**位置标识规范（必须使用）：**
`顶部操作栏 / 首屏信息区 / 汇总数据区 / 关联商机列表 / 联系人模块 / 销售记录区 / 审批列表 / 合同订单区 / 页面导航 / 弹窗 / 表单区域 / 其他:{自定义}`

**严重性升级规则：**
任何发现直接违反 design-system-contract.md 中的明确规则，自动升级一级：
P2→P1，P1→P0。标注：`升级：是 — 违反 design-system-contract §{section}`

## A1 — 间距系统

只允许：4 / 8 / 12 / 16 / 24 / 32 / 40 px

- [ ] 相邻元素间距在允许值范围内
- [ ] 卡片内边距一致（同类卡片 padding 相同）
- [ ] 组间距 > 组内距（相关内容分组视觉清晰）
- [ ] 表单标签与输入框上下对齐一致
- [ ] 列表行间距统一，无随机差异

---

## A2 — 排版规范

- [ ] L1（15px/500/#181C25）：区块标题层级正确
- [ ] L2（13px/400/#181C25）：核心内容/字段值
- [ ] L3（13px/400/#91959E）：辅助标签/字段名
- [ ] L4（12px/400/#91959E）：弱信息/时间戳
- [ ] 标题层级视觉权重递减，不跳级
- [ ] 全页字重种类 ≤ 3 种
- [ ] 同级文本字号一致

---

## A3 — 颜色系统

- [ ] 品牌主色 #FF8000 全页使用 ≤ 3 处
- [ ] 状态色语义正确：红=错误/危险，黄=警告，绿=成功，蓝=信息
- [ ] 状态色不混用
- [ ] 背景/文字颜色取自设计规范色板，无随机自定义色值
- [ ] 禁用状态颜色统一（灰色系）

---

## A4 — 组件视觉一致性

- [ ] 主要按钮样式全页统一（同一 variant 不能有两种外观）
- [ ] 次要按钮、幽灵按钮样式各自统一
- [ ] 图标风格统一（线性不与填充混用）
- [ ] 输入框边框样式统一
- [ ] 表格表头/行/分隔线样式全页统一
- [ ] 卡片 shadow 与 border 二选一，不同时使用
- [ ] Badge/Tag 颜色语义一致

---

## A5 — AI Slop 检测（独立评分 0–100，不影响 UX 综合分）

从 100 分开始，发现以下模式各扣分：

| 模式 | 扣分 |
|------|------|
| 通用紫色/蓝色渐变背景（与品牌无关） | -25 |
| 千篇一律的三列图标卡片网格 | -20 |
| 无意义装饰性插图（空状态除外） | -15 |
| 所有容器 border-radius 相同 | -15 |
| 按钮文案模糊（「提交」「确认」「点击」） | -10 |
| 用 emoji 作为功能性图标 | -15 |
