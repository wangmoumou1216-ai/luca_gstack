# UX 评审框架

模块A：视觉系统合规（是否遵守设计规范，不判断可用性）
模块C：CRM 业务专项（CRM 场景独有问题，不涉及通用可用性）

模块B（Nielsen/Norman/WCAG）由 `/ux-audit` 的 Module B specialist 执行，不在本文件。

---

## 模块A：视觉系统合规

**边界**：只检查视觉规范执行情况。颜色对比度属于无障碍（WCAG），
交给模块B。按钮交互一致性属于行为一致性，交给模块B H4。
本模块只管「像素级视觉规范是否被遵守」。

### A1 — 间距系统

只允许使用以下间距值：4 / 8 / 12 / 16 / 24 / 32 / 40 px。

- [ ] 相邻元素间距在允许值范围内
- [ ] 卡片内边距一致（同类卡片 padding 相同）
- [ ] 组间距 > 组内距（相关内容分组视觉清晰）
- [ ] 表单标签与输入框上下对齐一致
- [ ] 列表行间距统一，无随机差异

### A2 — 排版规范

- [ ] 正文字号 ≥ 13px（B2B 高密度扫描底线；字号档真值源见 html-prototype-tokens.md `text-13`，14px 为禁用档）
- [ ] 标题层级视觉权重递减（H1 > H2 > H3，不得跳级）
- [ ] 全页字重种类 ≤ 3 种（Regular / Medium / Bold）
- [ ] 行高 1.4–1.6（正文）；标题行高 1.2–1.4
- [ ] 同级文本字号一致（同为次级标签则字号相同）

### A3 — 颜色系统

- [ ] 品牌主色 #FF8000 全页使用 ≤ 3 处
- [ ] 状态色语义正确：红=错误/危险，黄=警告，绿=成功/完成，蓝=信息
- [ ] 状态色不混用（绿色按钮不用于危险操作）
- [ ] 背景/文字颜色取自设计规范色板，无随机自定义色值
- [ ] 禁用状态颜色统一（灰色系，不使用其他颜色表示禁用）

### A4 — 组件视觉一致性

- [ ] 主要按钮样式全页统一（同一 variant 不能有两种外观）
- [ ] 次要按钮、幽灵按钮样式各自统一
- [ ] 图标风格统一：线性图标不与填充图标混用
- [ ] 输入框边框样式统一（不能部分有边框部分无边框）
- [ ] 下拉框、日期选择器、多选框视觉风格一致
- [ ] 表格表头/行/分隔线样式全页统一
- [ ] 卡片投影（shadow）与边框（border）二选一，不同时使用
- [ ] Badge/Tag 组件颜色语义一致（相同含义用相同颜色）

### A5 — AI Slop 检测（独立评分 0–100）

从 100 分开始，发现以下模式各扣分：

| 模式 | 扣分 | 说明 |
|------|------|------|
| 通用紫色/蓝色渐变背景 | -25 | 与品牌无关的装饰性渐变 |
| 千篇一律的三列图标卡片网格 | -20 | 无内容差异的卡片平铺 |
| 无意义装饰性插图（空状态除外） | -15 | 与功能无关的 illustration |
| 所有容器 border-radius 相同 | -15 | 按钮/卡片/输入框 radius 一模一样 |
| 按钮文案模糊（「提交」「确认」「点击」） | -10 | 应具体描述操作（「新建商机」「保存记录」） |
| 用 emoji 作为功能性图标 | -15 | 替代正式图标库 |

AI Slop Score 不影响 UX 综合评分，单独列出。

## 模块C：CRM 业务专项

**边界**：只检查 CRM 场景独有的问题。「操作效率」在模块B H7
已有通用覆盖，本模块只检查 CRM 特定高频操作。「信息密度」在模块B H8
已有通用覆盖，本模块只检查 CRM 必须首屏可见的业务字段。

### C1 — CRM 核心字段首屏可见性

以下字段必须在 800px 视口高度内不需要滚动即可见（适用于客户/商机/联系人详情页）：

- [ ] 客户名称 / 商机名称
- [ ] 当前阶段 / 成交状态
- [ ] 负责人
- [ ] 最后跟进时间
- [ ] 商机金额（商机页）/ 客户等级（客户页）
- [ ] 快捷操作按钮区（新建跟进记录、拨打电话、发送邮件）

以上任何一项不可见 → P0 问题。

### C2 — CRM 高频操作路径深度

以下操作从当前页面出发，点击次数不得超过 2 次：

- [ ] 新建销售跟进记录
- [ ] 推进商机阶段
- [ ] 新建联系人
- [ ] 发起审批
- [ ] 查看关联合同

超过 2 次点击 → P1 问题。入口完全不存在 → P0 问题。

### C3 — 数据可信度（CRM 专属）

- [ ] 每个数据字段显示最后更新时间，或有明确的数据新鲜度标识
- [ ] 金额字段单位明确（元/万元不混用）
- [ ] 日期格式全页统一（YYYY-MM-DD 或 MM/DD/YYYY，不混用）
- [ ] 空字段有占位文本（「暂无」「未填写」），不显示空白或 null
- [ ] 异常数据有视觉标记（超期未跟进、金额异常等）
- [ ] 数字超过万位有千分位分隔符

### C4 — CRM 列表页专项（适用于列表/看板页）

- [ ] 列表支持按关键字段排序（金额、时间、阶段）
- [ ] 筛选条件保持状态（离开页面再回来，筛选不重置）
- [ ] 批量操作入口在选中后才出现（不占用常驻空间）
- [ ] 空列表状态有明确提示和引导操作（不显示纯空白）

---

## 优先级定义

- **P0**：严重阻碍用户完成核心 CRM 任务，必须修复
- **P1**：明显影响效率或可用性，应该修复
- **P2**：体验优化项，有余力时修复
