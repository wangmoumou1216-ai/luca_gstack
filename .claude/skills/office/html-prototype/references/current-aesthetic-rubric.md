# Current B2B SaaS / AI UI Aesthetic Rubric

> 被 `/html-prototype` 必读。用途：保证原型不只“符合 token”，
> 还符合当前一线 B2B SaaS / AI Native 产品审美。

## 0. 核心判断

当前优秀 B2B SaaS / AI UI 的目标不是炫技，而是：

```text
confidence > complexity
```

用户应该更快判断、更少犹豫、更清楚下一步，而不是看到更多装饰和更多功能入口。

## 1. 参考产品坐标

生成前用这些产品作为审美坐标，不复制外观，只借鉴决策方式：

| 产品/系统 | 借鉴点 | 禁止误读 |
|---|---|---|
| Linear | 冷静、密度克制、层级清楚、操作路径短 | 不要复制暗色科技感 |
| Attio | CRM 对象关系、属性字段、列表密度、可编辑信息结构 | 不要把所有对象做成营销卡片 |
| Notion / Notion AI | AI 低侵入、内容优先、渐进披露 | 不要把 AI 包成彩色装饰标签 |
| Granola | AI 结果像工作流里的自然层，不抢主任务 | 不要用“AI 魔法”视觉隐喻 |
| Cursor | Agent 执行可见、可暂停、可接管、可回退 | 不要隐藏长任务过程 |
| Atlassian Design | 企业组件清晰、反馈明确、状态和消息克制 | 不要堆复杂浮层 |
| Vercel / v0 / shadcn | token 化、组件化、小步迭代、可复用 UI primitives | 不要变成通用 shadcn 默认皮肤 |
| Salesforce Agentforce | AI 嵌入业务对象和流程，而不是独立聊天框 | 不要把 AI 做成旁边的万能助手 |

## 2. 2026 B2B SaaS 审美原则

### 2.1 Calm Density

界面应有信息密度，但不能躁：
- 表格、字段、动态流可以密集。
- 操作按钮必须收敛。
- 每屏只允许 1 个主要视觉重心。
- 卡片只用于真实分组，不用作装饰。

### 2.2 Invisible AI

AI 应该像基础设施，而不是舞台主角：
- AI 建议靠位置、徽标、弱提示表达来源。
- 只有需要用户信任/接管时，AI 状态才升层。
- 避免“✨ AI Suggestion”“魔法渐变”“机器人头像”。

### 2.3 Progressive Disclosure

复杂能力必须分层出现：
- 默认只展示完成当前任务需要的信息。
- 高级选项进入折叠、更多、抽屉或二级动作。
- 不把所有状态、解释、配置同时铺开。

### 2.4 Role / Context Awareness

CRM 用户不是抽象用户。原型必须说明：
- 销售当前最想知道什么？
- 管理者当前最想比较什么？
- 运营/管理员当前最怕误操作什么？

不同角色不能看到同样的信息优先级。

### 2.5 Trust Before Delight

B2B AI 体验的第一审美是可信：
- 给出来源、置信度、更新时间、可撤销入口。
- 低置信不要红色恐吓，用灰色降权 + 解释。
- Agent 执行中必须显示当前步骤、进度、暂停/接管。

## 3. 视觉评分表

生成 HTML 前后都用这张表自评。低于 24/30 不得交付。

| 维度 | 0 分 | 1 分 | 2 分 | 3 分 |
|---|---|---|---|---|
| 信息层级 | 无主次 | 有主次但混乱 | 主次清楚 | 用户一眼知道先看哪里 |
| 视觉密度 | 空泛或拥挤 | 局部失衡 | 基本合理 | 像真实高频 B2B 工具 |
| AI 融入 | 装饰化/魔法化 | AI 抢主任务 | AI 辅助主流程 | AI 像自然工作流层 |
| 信任机制 | 无来源/无状态 | 有提示但弱 | 有来源/置信/状态 | 可见、暂停、接管、撤销完整 |
| 渐进披露 | 全部铺开 | 部分折叠 | 默认层级合理 | 复杂度随任务逐步展开 |
| 组件规范 | 随意拼装 | 局部不一致 | 基本符合 framework | 像同一产品原生页面 |
| 当前审美 | 过时/AI 模板 | 有明显模板感 | 基本现代 | 可放进一线 SaaS 产品截图集 |
| 文案真实感 | Lorem/空泛 | 泛 B2B | CRM 真实 | 符合销售/管理者语境 |
| 状态完整度 | 只有默认态 | 基础状态不全 | 基础状态全 | AI/Agent 状态也完整 |
| 交互可理解 | 按钮无反馈 | 部分反馈 | 主要动作有反馈 | 用户可预期每个动作后果 |

## 4. 必须避免的“看起来像 AI 生成”

- Hero 化后台页面。
- 所有内容卡片化。
- 圆角和阴影过度。
- 灰紫蓝渐变。
- 大面积插画空态。
- “AI”到处高亮。
- 表格变卡片墙。
- 按钮文案只有“开始”“提交”“确认”，没有业务语义。
- 数据像“客户A/客户B/任务1”。
- 页面没有真实角色和真实任务。
