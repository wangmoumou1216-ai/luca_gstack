> 消费者：/research-kit skill。定位速记：**kit 武装 luca 去采集，insight-synthesis 消化采回来的数据**——本文件只管工具设计，不管发现与解读。

## 1. 访谈提纲（Interview Guide）

结构（60 分钟版，按时长等比缩放）：**开场**（2-3min：目的/预期/录音同意）→ **暖场**（3-5min：背景与角色的低门槛问题）→ **核心探索**（20-30min：按研究主题组织的深挖问题 + 追问预案）→ **具体场景走查**（10-15min：JTBD 式还原——"上次…是什么时候？当时想完成什么？接下来发生了什么？"）→ **收尾**（3-5min：有没有我没问到的？下一步/致谢）。
- **漏斗原则**：宽背景问题在前，具体功能问题在后；一次访谈聚焦 2-3 个研究主题。
- **非引导性守则（硬门，每个问题过一遍）**：问题含预设答案/预设情绪即重写。

| 引导性（禁） | 非引导（用） |
|---|---|
| "那是不是很麻烦？" | "你当时感觉如何？" |
| "好用吗？" | "对你来说难易程度如何？" |
| "你喜欢这个功能吗？" | "你注意到这个功能的什么，如果有的话？" |
| "你会用这个吗？" | "如果有的话，你会在工作里怎么用它？" |

- 能用是/否回答的问题一律扩成开放式（补"…为什么"）；不问假设性问题（"如果有 X 你会…"），问已发生的行为。
- 附**主持人备注**：保持中立、处理跑题、时间管理提示。

## 2. 问卷设计（Survey）

- **先判用途**：问卷做定量（多少人有此需求/态度分布/随时间变化），**不做探索**——不知道问题存在时用访谈，问卷只确认与量化。
- 结构与顺序：目的+耗时声明开场（无引导性预框）→ 筛选/人口学（短，放最前）→ **行为问题先于态度问题** → 开放题放最后（防疲劳污染核心题）。
- 题型速查：互斥单选（选项穷尽+兜底"其他"）/多选（勿用于需排序场景）/Likert（**5 点足够、保留中点、两端必须有文字标注、全卷方向一致**）/排序（≤5-7 项，认知负担大）/开放题（省着用，分析昂贵）。
- 写题四禁：引导（"我们快速的物流"→删"快速"）/双管（"结账是否又快又好"→拆两题）/回忆过载（12 个月改更短周期）/行话（用用户的词）。
- 量表基准：NPS 0-10（9-10 推荐者、0-6 贬损者，单指标勿当完整满意度）；SUS 十题原文照用不改（均值 68，>80 良好）。
- 纪律：**每题先答"这题回答什么决策"**，答不出就删；<5 分钟完卷，>10-15 题质量陡降；发出前 3-5 人认知预测试；报告结果必须带样本量与回收率，Likert 报分布不只报均值。

## 3. 可用性测试计划（Usability Test Plan）

七件套：**研究问题**（3-5 个具体可答的）→ **方法**（有/无主持、远程/现场、think-aloud）→ **被试**（筛选标准、每细分 5-8 人、招募渠道）→ **任务**（5-8 个真实场景任务，每个带成功判据+预期时长；任务给情境不给操作指令——"给这位新客户记一条跟进"而非"点击右上角按钮"）→ **指标**（完成率/用时/错误率/SEQ 或 SUS/主观满意度）→ **主持脚本**（开场/放任务/中立追问/收尾复盘）→ **分析计划**（发现如何综合与排优先级——产出直接喂 insight-synthesis）。
- 附**试运行清单**：正式场次前用 1 人跑通（原型可达、任务措辞无歧义、录制正常、时长合理）。

## 4. 卡片分类（Card Sorting → 树测试）

- **开放式**（被试自建分组自命名）用于探索新结构；**封闭式**（给定类目归卡）用于验证既有结构。15-30 张卡、15+ 人可出稳定模式。
- 分析四步：常见分组模式与命名习惯 → **相似度矩阵**（每对卡被归为一组的频率）→ 据用户心智模型提出导航类目建议 → 标记归类不一致的**含混项**（它们是 IA 的风险点，不是噪音）。
- 类目命名从用户词汇取（卡片分类与访谈原话），不用内部行话。
- **下一步永远是树测试**：新结构建导航组件前先树测（廉价，且能在构建前暴露 findability 失败）；关键任务上补首点击测试。
