# 翻译质量细则（muse-x-digest / 【一 · 完整翻译】）

> 目标：忠实、无遗漏、读起来像**人写的自然中文**，不是机翻腔。

## 1. No-omission 锚点（最高优先）
- **每一个英文句子都要有对应的中文句子。** 事实、数字、逻辑、专有名词必须完全对应，
  不概括、不跳过、不润色添加。（借 cuimao-translator）
- 自检四类错误（借 andrewyng/translation-agent 的准确度轴）：
  **addition（无中生有）/ omission（漏译）/ mistranslation（错译）/ untranslated（残留没译的英文）**。
  收尾逐类扫一遍。
- 保留原文分段与逻辑顺序；标题、列表、编号步骤的结构照搬。

## 2. 欧化中文六红旗（出自然中文；借 cuimao 六红旗）
逐句自检，命中即改：
1. **被字滥用** → 能主动就主动（"被广泛使用" → "大家普遍在用"）。
2. **冗余连词**：中文靠语序意合，"因为…所以…""虽然…但是…"多数可删一半。
3. **长定语链**：一长串"……的……的……的 X"拆成短句。
4. **习惯性"之一"**："是最好的方法之一" → 视原文常可直接"是很好的方法"。
5. **单句 ≥3 个"的"** → 警示，重组。
6. **名词化压动词**："进行了一次优化" → "优化了一下"；多用动词、少用"进行/实现/作出+名词"。

## 3. 术语与品牌（借 feiskyer 白名单）
- **保留不译的通用技术词**（首次出现可加中文注但不替换）：AI、LLM、GPT、API、RAG、Transformer、
  Token、Prompt、Agent、Fine-tuning、context、prompt engineering 等。
- **品牌 / 产品名不动**：OpenAI、Anthropic、Claude、Claude Projects、Cowork、GitHub、X（Twitter）等。
- **产品功能名**：首次给"英文原名（中文直译）"，之后可用中文，如
  `custom instructions（自定义指令）`、`knowledge base（知识库）`、`project instructions（项目指令）`。
- ⚠️ 与【二 · 深度讲述】分工：翻译区**保留**术语原样；到讲述区才**解释**它们（别在翻译里塞解释）。

## 4. 长文一致性（借 DelTA / cuimao）
- 长文**顺序处理**，不并行分块（并行会坏术语/口吻一致性）。
- 心里维护一张「术语 → 固定译法」小表，全文同一术语同一译法（Project 全篇要么"项目"要么保留，别混）。

## 5. 图片 OCR 纪律
- 信息图 / 文字截图：先把图里**全部文字** OCR 出来再逐条翻译；并**说明图的结构逻辑**
  （谁指向谁、闭环/流程关系）。
- 纯装饰图 / 配图：一句话描述即可，不过度解读，不当正文。
- 图里读不出的地方如实说"此处图中文字不清晰"，不猜。
