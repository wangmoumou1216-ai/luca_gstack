# 按需披露 · 深度阶梯（muse-x-digest / 【二 · 深度讲述】）

> 核心：**同一个 skill，披露量随内容实质缩放。** 长文全量深读，短帖点到为止。
> 反面教材（避开）：无论内容多短都套同一套"深度分析"模板 → 注水。

## 定档（先判类型，再判档）
| 档 | 命中特征 | 讲述量 |
|---|---|---|
| **Micro** | 一句话观点 / 段子 / 纯转链 / 一张自足海报 | 忠实翻译 + **至多 1-2 行**：作者是谁（若非知名）、指代的东西是什么。**不做深挖。** |
| **Standard** | 普通信息帖 / 几条短推的串 / 一个明确观点带简单论据 | 翻译 + **紧凑解读**：关键术语一句话、作者核心观点、必要背景。 |
| **Deep** | 长文(Article) / 教程 / 结构化论述 / 多步骤方法 | 逐段翻译 + **完整深读**（下方结构）。 |

**拿不准 → 往低一档走**（宁简洁勿注水）。档位要在产出里**显式说一句**（如"这是一条短推，简要处理"）。

## Deep 档的讲述结构（借 project-teardown-skill）
1. **概览先行**：先给一句话判断（这篇到底在讲什么、值不值得看）→ 本质 → **为什么重要** → 关键机制清单。
2. **再只深挖最值的 2-4 点**，不逐段复述。挑选标准：读者不懂就 get 不到全文价值的那几点。
3. **why-over-what**：解释每个点用"**没有它会怎样 / 它解决了什么问题**"，而不是罗列"它有什么功能"。
4. 讲透：术语与缩写、必要背景、人物/产品/事件指代、作者真正的观点、以及它为什么值得注意。

## 反膨胀铁律（借 Yirzzzz 原话精神）
- **相关但会稀释主线的内容，不硬塞。** 宁可少讲一个次要点，也别让主线糊掉。
- 短帖不补它不需要的背景。判断标准：这段背景**删掉**会让读者看不懂吗？不会 → 删。
- 讲述是为了让读者"不看英文也完全懂这条在讲什么"，不是为了显得详尽。
